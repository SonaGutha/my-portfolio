Mini Case Studies

Centralized Event Logging Platform | fAIshion.ai

Problem: Logging was service-specific, MongoDB-heavy, and caused duplicate calls and schema drift.

Approach: Built a centralized logging API and standardized event schema across backend and frontend integrations.

Tradeoffs: Invested upfront in shared contracts and migration effort to gain long-term observability and consistency.

Result: Reduced redundant DB calls by ~60% and improved reliability of production event capture.

Health Insurance Modernization | Deloitte

Problem: Legacy mainframe workflows were limiting scalability and slowing claims turnaround.

Approach: Migrated logic into Java/Spring Boot microservices with REST APIs and IBM ODM rule services.

Tradeoffs: Balanced strict release timelines with phased modernization to reduce migration risk.

Result: Delivered 65+ successful sprints, improved processing efficiency, and maintained zero post-release defects.

Life Insurance API Enablement | TCS

Problem: Legacy life insurance modules needed API-first access and cleaner maintainability.

Approach: Developed RESTful Java services, optimized backend modules, and supported stable releases with unit tests.

Tradeoffs: Worked within legacy constraints while incrementally introducing cleaner coding patterns.

Result: Improved service scalability, reduced production issues, and increased delivery confidence.
